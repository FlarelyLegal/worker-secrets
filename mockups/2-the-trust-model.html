Secret Vault

What happens when the server is compromised?

Most secret managers protect secrets at rest. This one protects them from the server itself. Your secrets are encrypted on your machine before they leave it. The server stores ciphertext it cannot decrypt.

Threat model

If an attacker gets... Standard secrets E2E secrets
Database access (D1) Encrypted Needs ENCRYPTION_KEY to decrypt Ciphertext inside ciphertext Also needs your age private key
ENCRYPTION_KEY Can decrypt all Still only age ciphertext
Full Worker compromise Can read at runtime Server never had plaintext
Cloudflare account Dashboard shows master key Your private key is on your laptop
Your laptop N/A (no secrets stored locally) Identity file decrypts your secrets
The only way to read an e2e-private secret is to have your age identity file and the API credentials to fetch the ciphertext. Compromise one without the other and you get nothing useful.

Deploy

hfs keygen --register && hfs set DB_PASSWORD "value" --private
