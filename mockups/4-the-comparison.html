Secret Vault

The vault that doesn't trust itself.

Hosted secret managers can read your secrets. Self-hosted ones require servers to maintain. This runs on Cloudflare Workers with zero-knowledge encryption. No servers. No trust.

Feature Secret Vault Hosted vaults Self-hosted
Server can read secrets No (with e2e) Yes Yes
Infrastructure to maintain None (serverless) None (hosted) VMs, updates, backups
Third-party data access Your Cloudflare account only Vendor has access Self-hosted
Per-secret encryption keys Envelope encryption Varies Varies
Client-side encryption age (private or team) No No
RBAC-based recipient sharing Automatic from roles N/A N/A
Geo-fencing by country Built-in (Cloudflare edge) No No
Burn-after-reading Flag + tag No No
Tamper-evident audit SHA-256 hash chain Append-only logs Varies
Deploy time One command Signup + config Hours to days
Cost Cloudflare free tier Per-seat pricing Server costs

Other vaults encrypt secrets at rest. This one encrypts them before they reach the server. Your private secrets exist as ciphertext the server cannot decrypt. Team secrets are encrypted for specific people based on their roles. Remove someone, rewrap, and their key is excluded. The trust boundary is your machine, not a cloud service.

# Five minutes from nothing to zero-knowledge
$ npm i -g @FlarelyLegal/hfs-cli --registry=https://npm.pkg.github.com
$ hfs deploy
$ hfs login
$ hfs keygen --register
$ hfs set DB_PASSWORD "hunter2" --private
