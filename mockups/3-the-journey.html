S
Secret Vault

From zero to zero-knowledge in four steps.

Deploy an encrypted secret vault to your Cloudflare account, store secrets the server cannot read, and share them with your team. No external dependencies. No third-party trust.

1
Deploy
One command scaffolds a Cloudflare Worker, D1 database, KV namespace, and Access policies. The vault runs on your account.
$ npm i -g @FlarelyLegal/hfs-cli --registry=https://npm.pkg.github.com
$ hfs deploy
Deployed to https://secrets.yourcompany.com
2
Authenticate
Log in through your identity provider via Cloudflare Access. Hardware keys supported. Your session is a short-lived JWT, never stored on the server.
$ hfs login
Authenticated successfully
Session expires: 3/30/2026, 2:25 PM
3
Store
Generate your age identity and store secrets. Use --private for secrets only you can decrypt, or --e2e to share with teammates whose roles match.
$ hfs keygen --register
public key: age196nua3eeww...

$ hfs set API_KEY "sk-ant-..." --private
Stored API_KEY (e2e private)

$ hfs set DEPLOY_TOKEN "ghp_..." --e2e -t ci
encrypting for 4 recipients
Stored DEPLOY_TOKEN (e2e)
4
Use everywhere
Load secrets into your shell, CI pipelines, config files, or any tool that reads environment variables. E2E secrets are decrypted locally, never on the server.
# Shell
$ eval $(hfs env -e API_KEY DB_PASSWORD)

# Config files
$ hfs template .env.tpl > .env

# GitHub Actions
- uses: FlarelyLegal/worker-secrets/action@v0.20.0
End-to-end encryption
Private or team-shared. age encryption. Server never sees plaintext.
RBAC with tags
Roles control scopes. Tags control which secrets. Recipients follow automatically.
Burn after reading
One-time secrets that self-destruct after first read.
Geo-fencing
Restrict access by country. Only possible on Cloudflare Workers.
Webhooks
Audit events to Slack, PagerDuty, or your SIEM. Zero latency impact.
23 feature flags
Runtime control without redeploying. Maintenance mode, read-only, expiry enforcement.
Version history
Every change archived. Diff against versions. Restore with one command.
Tamper-evident audit
SHA-256 hash-chained log. Verifiable with the CLI.
